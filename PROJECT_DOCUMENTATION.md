# Документация проекта "Pinch & Drop"

## Общее описание
Проект представляет собой полнофункциональное веб-приложение для презентации и управления каталогом продукции бренда "Pinch & Drop" (сиропы и пюре для профессиональной миксологии). Приложение состоит из клиентской части (Landing page + Admin панель), сервера на Node.js и базы данных PostgreSQL. Отличительной чертой проекта является использование иммерсивного 3D-фона и плавных анимаций для создания премиального пользовательского опыта.

## Технологический стек

### Frontend (Клиентская часть)
*   **Фреймворк:** React 19
*   **Сборщик:** Vite
*   **Маршрутизация:** React Router DOM v7
*   **3D Графика:** Three.js, @react-three/fiber, @react-three/drei (интерактивный фон с частицами и искажающейся сферой)
*   **Анимации:** GSAP (ScrollTrigger), Framer Motion
*   **Скролл:** Lenis (плавная прокрутка)
*   **Иконки:** Lucide React
*   **Стилизация:** Чистый CSS (`src/index.css`)

### Backend (Серверная часть)
*   **Среда выполнения:** Node.js
*   **Фреймворк:** Express.js
*   **База данных:** PostgreSQL
*   **Драйвер БД:** `pg` (node-postgres)
*   **Middleware:** `cors` (для междоменных запросов), `express.json` (для парсинга JSON).
*   **Конфигурация:** `dotenv` (переменные окружения из файла `.env`).

## Архитектура и функционал

### 1. Клиентская часть (`src/App.jsx`)
Документ `App.jsx` содержит всю концептуальную логику фронтенда, разделенную на несколько ключевых компонентов:
*   **BackgroundScene:** Three.js сцена с частицами и анимированной сферой, реагирующая на скорость прокрутки страницы.
*   **Landing:** Основной экран для пользователей сайта. Содержит:
    *   Главный экран (Hero) с 3D фоном.
    *   Навигацию по категориям.
    *   Сетку товаров (Bento Grid) с интерактивными эффектами при наведении курсора (наклон карточек).
    *   Блок контактов и форму обратной связи.
    *   Модальное окно просмотра товара.
*   **Admin:** Панель администратора (доступна по маршруту `/admin`). Позволяет:
    *   Изменять заголовки и слоганы главного экрана.
    *   Управлять каталогом: добавлять/удалять категории и товары.
    *   Просматривать список заявок (в текущей реализации функционал в разработке/заглушка).
    *   Просматривать настройки контактов.
*   **MainApp:** Корневой компонент, осуществляющий загрузку данных с сервера (`GET /api/data`) перед рендерингом приложения, а также предоставляющий функцию обновления данных (`POST /api/data`).

### 2. Серверная часть (`server.js`)
Реализовано два основных API эндпоинта:
*   `GET /api/data`: Извлекает настройки сайта, категории и товары из базы данных PostgreSQL и формирует их в единый JSON-объект, структура которого ожидается клиентом.
*   `POST /api/data`: Принимает обновленные данные из панели администратора. Обновляет данные в таблице `site_settings` и полностью перезаписывает таблицы `categories` и `items` (упрощенный подход для быстрого прототипирования). Все операции обернуты в транзакцию (`BEGIN`, `COMMIT`, `ROLLBACK`) для предотвращения частичного обновления в случае ошибки.

### 3. База данных (`schema.sql`)
Используется 4 таблицы:
*   `site_settings`: Общие настройки сайта (бренд, слоган, контакты).
*   `categories`: Категории товаров (ID, заголовок, мета-описание).
*   `items`: Товары, привязанные к категориям (связь через `category_id`).
*   `requests`: Журнал заявок пользователей (Имя, Телефон, дата создания).

### 4. Миграция (`migrate.js`)
Одноразовый скрипт, предназначенный для переноса исходного состояния данных, ранее хранимого в `db.json`, в реляционную структуру PostgreSQL. Очищает целевые таблицы перед вставкой новых данных для предотвращения дублирования.

## Запуск проекта

1.  **База данных:** Убедитесь, что у вас установлен, запущен PostgreSQL и создана база данных `pinch_db` с пользователем `pinch_user` (пароль `pinch_pass`), как указано в `.env`. Структура базы инициализируется через `schema.sql`.
2.  **Миграция данных (при первом запуске):**
    ```bash
    node migrate.js
    ```
3.  **Запуск Backend сервера (Express):**
    ```bash
    node server.js
    ```
    Сервер запустится на `http://localhost:5000`.
4.  **Запуск Frontend сервера (Vite):**
    В отдельном окне терминала выполните:
    ```bash
    npm run dev
    ```
    Клиентское приложение будет доступно по адресу `http://localhost:5173`. Открыть админ-панель можно перейдя по адресу `http://localhost:5173/admin`.
